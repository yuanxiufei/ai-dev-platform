### MenuBar 下拉菜单完全重写 (20:18)
- **根因**: 下拉菜单也有 4 个层面同时出问题
  1. `$el` 在 Vue3 模板中不可用 → `@mouseenter`/`@mouseleave` 的 TS 类型断言全报错
  2. 所有 Tailwind 类不渲染: `min-w-[220px]`, `py-1`, `px-4`, `py-1.5`, `gap-8`, `text-[14px]`, `leading-7`, `w-full`, `text-left`, `flex`, `justify-between`, `items-center`
  3. `:class` 动态类 `text-[var(...)]` 不生效
  4. hover-toggle 逻辑错误: 鼠标悬停同一菜单时 toggleMenu 会关闭它
- **修复**:
  - 容器 `class="absolute top-full..."` → inline `position:absolute; top:100%; left:0; min-width:220px; padding:4px 0; z-index:100`
  - 菜单项按钮全部 inline style: `width:100%; textAlign:left; padding:6px 16px; display:flex; justifyContent:space-between; fontSize:14px; lineHeight:28px`
  - 动态 checked 颜色: `color: item.checked ? 'var(--color-ide-accent)' : 'var(--color-ide-text)'` 直接在 `:style` 中
  - hover 效果: `@mouseenter="(e) => { e.currentTarget.style.background = 'var(--color-ide-menu-hover, #094771)'; ... }"`
  - TS 类型移除: `(e: MouseEvent)` → `(e)`, `as HTMLElement` → 直接用 `e.currentTarget.style`
  - 菜单按钮 `@mouseenter`: `openMenu && toggleMenu(menu.id)` → `openMenu && openMenu !== menu.id && (openMenu = menu.id)` — 只切换不同菜单，不误关
  - 菜单按钮本身: `:class` + Tailwind hover → inline `@mouseover`/`@mouseout` 操作 `e.currentTarget.style`
  - AI 助手按钮: `hover:text-*/hover:bg-*` → `@mouseover`/`@mouseout` inline
  - 容器: `class="flex shrink-0..."` → `display:flex; flex-shrink:0; user-select:none; z-index:50`
  - `class="relative"` → `style="position:relative"`
  - `class="flex-1"` → `style="flex:1"`
  - `class="flex items-center gap-0.5 px-1"` → `display:flex; align-items:center; gap:2px; padding:0 4px`
- 零 lint 错误，全部走 inline style 绕过 Tailwind
